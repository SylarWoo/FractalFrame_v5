# Configuration

## Backend Bridge

Start command:

```powershell
.\.venv\Scripts\python.exe scripts\mt5_symbols_server.py --host 127.0.0.1 --port 8765
```

Arguments:

- `--host`: listen address, default `127.0.0.1`.
- `--port`: listen port, default `8765`.
- `--cache-root`: MT5 symbol cache root.
- `--store-root`: StoreV5 data root. When omitted, the project runtime default is used.

Environment variables:

- `FRACTALFRAME_CORS_ORIGIN`: CORS origin, default `*`.
- `FRACTALFRAME_LOG_LEVEL`: log level, default `INFO`.
- `FRACTALFRAME_LOG_ROOT`: log directory, default `runtime_data/logs`.
- `FRACTALFRAME_JOB_SNAPSHOT_ROOT`: job snapshot directory, default `runtime_data/jobs`.

## Frontend

See `frontend/.env.example`:

```env
VITE_FRACTALFRAME_MARKET_DATA_HTTP_BASE=http://127.0.0.1:8765
```

## Start Scripts

```powershell
.\scripts\start_backend.ps1
.\scripts\start_frontend.ps1 -OpenBrowser
.\scripts\start_all.ps1 -OpenBrowser
```

## npm Warning

If npm prints:

```text
npm warn Unknown env config "min-release-age"
```

it usually comes from the parent environment variable `NPM_CONFIG_MIN_RELEASE_AGE`, not from this project. `scripts/check_all.ps1` clears it within its own process.
