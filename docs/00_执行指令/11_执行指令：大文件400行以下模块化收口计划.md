## 二、最终目标

1. 自有主要源码文件原则上压到 400 行以下。
2. 不改变现有功能、接口、localStorage key、用户数据结构和 UI 主流程。
3. 拆分后的模块要有清晰职责：
   - 容器组件只组合状态和子组件。
   - hook 管状态、副作用、事件源、异步任务。
   - util 管纯函数。
   - panel/component 管局部 JSX。
   - CSS 按面板或功能区拆分。
4. 每完成一个阶段都必须运行验证命令。
5. 修复乱码时优先使用 ASCII-safe 的 Unicode 转义写源码，例如：

```tsx
{'\u4ea4\u6613\u54c1\u79cd'}
```

避免在 PowerShell 管道里直接写中文，防止再次把源码写成问号或乱码。

## 三、统一执行原则

1. 每次只拆一个明确模块，不在同一阶段同时拆多个无关区域。
2. 先抽“纯函数、常量、类型、低耦合子组件”，再抽复杂 hook。
3. 拆 hook 时，先保持原状态名和原行为，再考虑整理命名。
4. 不做视觉重设计，不调整布局尺寸，除非是为了修复明显乱码或溢出。
5. 不删除用户数据、运行时数据、缓存数据。
6. 不回滚用户已有改动；如果工作区已有无关改动，忽略它。
7. 每阶段完成后记录：
   - 新增/修改文件
   - 原文件行数变化
   - 验证命令结果
   - 是否有残留风险

```powershell
Get-ChildItem -Path frontend,scripts,python,tests -Recurse -File |
  Where-Object {
    $_.FullName -notmatch '\\(node_modules|dist|__pycache__|\.git)\\' -and
    $_.Extension -in '.ts','.tsx','.py','.css'
  } |
  ForEach-Object {
    $count=(Get-Content -LiteralPath $_.FullName).Count
    [pscustomobject]@{
      Lines=$count
      Path=$_.FullName.Substring((Get-Location).Path.Length+1)
    }
  } |
  Where-Object { $_.Lines -gt 400 } |
  Sort-Object Lines -Descending |
  Format-Table -AutoSize
```
