# 5. 后端进度百分比规则

## 5.1 refresh 模式

refresh 有明确 `count`，所以百分比按目标总数算：

```py
readPercent = rowsFetched / maxCount
writePercent = rowsWritten / maxCount
```

但为了让读取和写入都能动，建议拆权重：

```txt
读取阶段：0% ~ 70%
写入阶段：70% ~ 96%
finalizing：96% ~ 99%
completed：100%
```

计算方式：

```py
if phase == "fetching":
    progressPercent = min(70, (rowsFetched / maxCount) * 70)

if phase == "writing":
    progressPercent = 70 + min(26, (rowsWritten / maxCount) * 26)

if phase == "finalizing":
    progressPercent = 98

if phase == "completed":
    progressPercent = 100
```

注意：

```txt
不要让 progressPercent 超过 99，除非 completed。
```

---

## 5.2 incremental 模式

incremental 不一定有明确总数。

如果没有 maxCount：

```txt
进度条用 indeterminate 或显示 45% 动态占位。
```

但是如果现在 UI 不支持 indeterminate，就按阶段给：

```txt
queued       2%
fetching     15% ~ 65%，按 chunksCompleted 逐步增加，但最多 65%
writing      70% ~ 90%
finalizing   98%
completed    100%
```
