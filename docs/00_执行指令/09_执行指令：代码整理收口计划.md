# 执行指令：FractalFrame_v5 代码整理收口计划

> 这份文档是给后续执行者看的工作计划。目标不是重新设计系统，而是在现有功能已经跑通的基础上，把设置模块、MT5 数据中心、图表宿主、后端 HTTP 桥做一次低风险收口，避免后续继续加功能时文件膨胀、状态散落、职责混乱。

## 当前基线

当前已完成并可工作的模块：

1. 前端工作台主图：`frontend/src/workbench/chart/ChartCoreHost.tsx`
2. 右侧设置面板：`frontend/src/workbench/rightDrawer/RightDrawer.tsx`
3. MT5 数据中心 UI：`frontend/src/workbench/rightDrawer/RightDrawer.tsx`
4. StoreV5 前端 API：`frontend/src/workbench/rightDrawer/mt5SymbolsApi.ts`
5. StoreV5 后端 HTTP 桥：`scripts/mt5_symbols_server.py`
6. StoreV5 数据仓库核心：`python/data_warehouse/**`
7. StoreV5 测试：`tests/test_store_v5.py`

当前验证命令：

```powershell
cd G:\PythonProject\FractalFrame_v5
.venv\Scripts\python.exe -m unittest tests.test_store_v5

cd G:\PythonProject\FractalFrame_v5\frontend
npm run build
```

执行任何整理前后，都必须跑这两条验证命令。

## 总目标

把代码整理成下面的状态：

1. 设置页面是独立模块，不被 MT5 数据中心逻辑污染。
2. MT5 数据中心是独立模块，业务状态、任务进度、SSE、持久化集中管理。
3. 图表宿主只负责图表生命周期、数据加载、图表样式应用，不承载过多调试和业务 UI 状态。
4. 后端 HTTP 桥只负责路由入口和响应发送，具体任务、SSE、StoreV5 操作拆到独立模块。
5. 所有设置 key 集中定义，避免字符串散落。
6. 先做无行为变化重构，再做小修复。每一步都可单独回滚。

## 禁止事项

1. 不重写 UI。
2. 不换后端框架。
3. 不改 StoreV5 文件格式。
4. 不改 API 路径，除非同时更新前端调用和文档。
5. 不删除 `runtime_data` 里的用户数据。
6. 不做大范围样式重构。
7. 不在同一步里同时改前端结构和后端结构。
8. 不为了拆文件改变用户可见行为。

## 第一阶段：清理低风险明显问题

### 1.1 修复乱码状态判断

位置：

```txt
frontend/src/workbench/rightDrawer/RightDrawer.tsx
```

当前问题：

```ts
storeActionStatus.includes('鎷夊彇')
```

这是中文“拉取”的乱码。它不应该作为业务状态判断依据。

执行要求：

1. 找到 `isPullingStoreV5`。
2. 改成只依赖结构化状态，例如：

```ts
const isPullingStoreV5 = storeCheckLoading && pullProgress != null
```

3. 如果 UI 仍需要区分拉取、检查、聚合，新增一个明确 helper，例如：

```ts
function resolveStoreOperationKind(...) {
  if (pullProgress) return 'pull'
  if (m1CheckJob) return 'check'
  if (aggregateProgress) return 'aggregate'
  return 'idle'
}
```

验收：

```powershell
npm run build
```

### 1.2 收口图表调试日志

位置：

```txt
frontend/src/workbench/chart/ChartCoreHost.tsx
```

当前问题：

文件里有多处：

```ts
console.info('[StoreV5Datafeed] ...')
console.warn('[StoreV5Datafeed] ...')
console.error('[StoreV5Datafeed] ...')
```

执行要求：

1. 新增小工具：

```txt
frontend/src/workbench/chart/chartLogger.ts
```

3. `info` 级别只在开发环境输出。
4. `warn` 和 `error` 可以保留输出，或者也受开关控制，但要统一。
5. 替换 `ChartCoreHost.tsx` 里的直接 console 调用。

验收：

