宝宝，熊熊看了 v4 的结构：v4 里 MT5 Import Center 是一个独立模块入口，安装 runtime + panel，并且它挂在右侧抽屉 slot 上，key 是 `mt5ImportCenter`，选择器是 `data-right-widget-panel="mt5ImportCenter"`。也就是说，v4 的思路本来就是“右侧业务面板插槽 + MT5 数据中心模块”这一套。
v4 的 panel 里也明确把 `MT5_IMPORT_CENTER_RIGHT_WIDGET_KEY_V1` 定成 `mt5ImportCenter`，然后通过 `mountShell` 把面板挂进去。

v5 当前 `App.tsx` 还比较干净：就是 `KLineChartPro` 初始化 + datafeed + 样式修正，没有右侧工作台层。 所以这刀可以很轻：**不碰图表内核，只在外层加右侧抽屉壳。**

# FractalFrame_v5 执行文案：新增右侧数据中心抽屉 V1

## 1. 本刀目标

在 FractalFrame_v5 当前 KLineChart Pro 页面右侧，新增一个类似 FractalFrame_v4 的右侧抽屉。

第一版只做：

```txt
右侧数据中心按钮
右侧抽屉打开 / 关闭
数据中心 SVG 图标
抽屉内部静态信息展示
不接复杂 MT5 操作
不改 KLineChart Pro 内核
不重建 chart
不重新加载 100万 M1 数据
```

本刀目的不是完整复刻 v4 的 MT5 Import Center，而是先验证：

```txt
v5 的 KLineChart Pro 页面 + 右侧 FractalFrame 抽屉
能不能共存
开关抽屉时会不会影响 100万 M1 K线拖动
视觉上是否适合继续迁移 v4 的 UI 管理层
```

---

## 2. 当前判断

v5 图表核心已经通过 100万根 M1 数据压测：

```txt
Bars: 1,000,000
Load: 约 1.8s
Ready: 约 2.2s
左右拖动不卡
MA / VOL 开启也不卡
```

所以当前不要再动图表数据链。KLineChart Pro 的 API 本身支持通过构造参数传入 `container / styles / symbol / period / datafeed`，也支持 `setStyles / setTheme / setTimezone / setPeriod` 等外部控制能力；这说明右侧抽屉以后可以作为 FractalFrame 自己的控制层，但这刀先不接复杂控制。([KLineChart Pro][1])

---

## 3. v4 参考原则

v4 的右侧业务面板模式可以继承思想，不要原样搬复杂代码。

v4 模式：

```txt
Right Drawer Slot
  ↓
data-right-widget-panel="mt5ImportCenter"
  ↓
MT5 Import Center Panel
  ↓
搜索栏 / 品种列表 / 导入计划 / 状态刷新
```

v5 第一版只借这几个原则：

```txt
1. 右侧抽屉是业务管理层
2. 数据中心是一个独立 panel
3. panel 通过明确 key 管理：mt5DataCenter
4. 图表内核不依赖抽屉
5. 抽屉开关不触发 chart 重新初始化
```

不要直接搬 v4 的 runtime、事件总线、symbol universe、lazy lower、splitter 那些复杂东西。v4 那些是完整 MT5 Import Center，不适合第一刀直接迁入。

## 6. App.tsx 改造要求

当前 `App.tsx` 里有：

```ts
const containerRef = useRef<HTMLDivElement | null>(null)

useEffect(() => {
  const chart = new KLineChartPro(...)
  return () => chart.destroy?.()
}, [])
```

这段逻辑保持不动。

只做外层 JSX 改造：

```tsx
return (
  <div className="ff-v5-root">
    <div className="ff-v5-chart-frame">
      <div ref={containerRef} className="ff-v5-chart" />
    </div>

    <RightWorkbenchDrawer />
  </div>
)
```

关键要求：

```txt
useEffect 依赖仍然是 []
RightWorkbenchDrawer 的 open/close state 不得进入 chart useEffect 依赖
RightWorkbenchDrawer 不传 bars 大数组
RightWorkbenchDrawer 不触发 createDatafeed
RightWorkbenchDrawer 不调用 chart.destroy
```

## 8. 抽屉布局

抽屉建议从右侧覆盖出来，不挤压图表。

```css
.ff-right-workbench-drawer {
  position: absolute;
  top: 38px;
  right: 0;
  bottom: 0;
  width: 360px;
  transform: translateX(100%);
  transition: transform 160ms ease;
  z-index: 20;
}

.ff-right-workbench-drawer[data-open="true"] {
  transform: translateX(0);
}
```

为什么用 overlay：

```txt
不改变 chart container 尺寸
不频繁触发 chart resize
开关抽屉时不影响 K线拖动
```
