1. 每次只处理一个明确阶段，不混合修 lint、拆模块、改测试框架等无关事项。
2. 优先修可验证问题，再做结构优化。

   - 不改 API path。
   - 不改 localStorage key。
   - 不改 StoreV5 数据结构。

5. CSS 和 UI 优化不做视觉重设计，只做测试覆盖、稳定性和可维护性增强。
6. 每阶段完成后必须记录：

## 四、阶段 1：修复 `npm run lint`

### 目标

让 `cd frontend; npm run lint` 返回 0，且不通过粗暴关闭规则来掩盖问题。

### 当前已知 lint 问题

```txt

frontend/src/workbench/rightDrawer/RightDrawer.tsx

  react-hooks/exhaustive-deps warning

frontend/src/workbench/chart/useChartInstance.ts
  react-hooks/exhaustive-deps warning
```

### 执行细则

1. `storeV5StatusFormat.ts`
   - 定位 `\x00` 控制字符正则。
   - 优先改为命名清晰的 helper，例如 `stripNulCharacters(value: string)`。
   - 如果仍需匹配 NUL 字符，避免触发 `no-control-regex`，可用字符码过滤：
     ```ts
     value.split('').filter((char) => char.charCodeAt(0) !== 0).join('')
     ```
   - 增加或更新 `storeV5StatusFormat.test.ts`，覆盖包含 NUL 字符的输入。

2. `RightDrawer.tsx`
   - 将 `useMemo(getInitialSymbolSnapshot, [])` 改为 inline function：
     ```ts
     const initialSnapshot = useMemo(() => getInitialSymbolSnapshot(), [])
     ```
   - `readSharedSelection` 同理。
   - 确认初始化只执行一次，不改变原有初始选中逻辑。

3. `useShortcutMenuState.ts`
   - 将 `useMemo(readShortcutMenuPeriods, deps)` 改为 inline function。
   - 对 effect 内同步 `setActivePeriod` 做行为检查：
     - 如果只是从 `periods` 派生 fallback，优先考虑在选择逻辑中处理。
     - 如果必须在 effect 中同步状态，尽量用事件触发或初始化函数消除 effect。
   - 增加/更新 `topbarPeriodUtils.test.ts` 或新增 hook 逻辑测试，覆盖 fallback period。

4. `useWatchlistRealtime.ts`
   - 先处理 refs/render 报错：
     - 不在 render 初始化闭包中读取可能被 lint 判断为 ref 的对象。
     - 如果 `initialRealtimeSnapshot` 是普通对象但规则误判，优先改为明确的局部常量或初始化 helper。
   - 再处理 effect 内同步 setState：
     - 将禁用实时行情时的 reset 行为收敛成显式函数。
     - 连接 EventSource 的 status 更新应尽量发生在事件回调或启动函数中。
     - 不为了消警告删除状态更新。
   - 保持以下行为不变：
     - watchlist realtime 开关状态。
     - EventSource 关闭时机。
     - log 最多条数。
     - foreground symbol tick 订阅。

5. `ChartCoreHost.tsx` / `useChartInstance.ts`
   - 检查 warning 是否是真依赖遗漏。
   - 如果 effect 依赖会导致图表实例重复创建，应将不稳定输入通过 ref 同步，而不是直接塞进依赖数组。
   - 如果加入依赖不会改变生命周期，则直接补齐依赖。

## 五、阶段 2：React hooks 状态流优化

### 目标

在 lint 通过的基础上，继续降低 hooks 内部复杂度，重点处理实时行情、快捷周期、图表实例生命周期。

### 重点文件

```txt
frontend/src/workbench/rightDrawer/useWatchlistRealtime.ts
frontend/src/workbench/rightDrawer/useStoreV5Jobs.ts
frontend/src/workbench/rightDrawer/useStoreV5M1CheckJobs.ts
frontend/src/workbench/topbar/useShortcutMenuState.ts
frontend/src/workbench/chart/useChartInstance.ts
frontend/src/workbench/chart/useChartRealtimeTicks.ts
frontend/src/workbench/chart/useChartStepLoad.ts
```

### 推荐拆分方向

1. `useWatchlistRealtime.ts`
   - 抽 `useWatchlistRealtimeLog`：
     - 管理 log push、截断、清空。
   - 抽 `useWatchlistTicksStream`：
     - 管理 watchlist EventSource。
     - 输出 ticks、ready、status。
   - 抽 `useForegroundTickStream`：
     - 管理单个 foreground symbol 的 tick EventSource。
   - 原 hook 只组合状态和对外动作。

2. `useStoreV5Jobs.ts`
   - 区分 job start、job polling/wait、result application。
   - 如果存在重复的 wait/poll 逻辑，继续收敛到 `storeV5JobWaiters.ts` 或新增共享 helper。

3. `useShortcutMenuState.ts`
   - 将 localStorage 读取、跨窗口 storage event、activePeriod fallback 拆开。
   - 避免 effect 里做可以由 reducer/initializer 完成的派生状态。

4. 图表 hooks
   - 图表实例创建/销毁只由容器生命周期控制。
   - symbol、period、displayName 等变化只走 update path。
   - 避免 effect 依赖导致实例重复销毁重建。

### 验收标准

1. 所有相关文件仍低于 400 行。
2. `npm run lint` 通过。
3. `npm run test:logic` 通过。
4. `npm run build` 通过。
5. Watchlist realtime、foreground tick、chart step load 的行为保持原样。

### 建议新增测试

1. realtime log 截断逻辑。
2. tick payload merge 逻辑。
3. shortcut period fallback 逻辑。

## 六、阶段 3：补浏览器层 smoke/E2E 回归

### 目标

当前逻辑测试已覆盖一部分纯函数，但 UI/CSS/面板拆分缺少浏览器层验证。本阶段目标是新增最小可维护的 smoke/E2E 回归，不追求大而全。

### 推荐工具

优先使用 Playwright。若项目暂不希望新增依赖，可先建立手动回归清单；但最终建议落到自动化。

### 建议命令

```powershell
cd G:\PythonProject\FractalFrame_v5\frontend
npm install -D @playwright/test
npx playwright install chromium
```

新增脚本：

```json
{
  "scripts": {
    "test:e2e": "playwright test"
  }
}
```

### 最小 smoke 用例

1. 应用能打开：
   - 访问 Vite dev server 或 preview server。
   - root 节点渲染成功。

2. 右侧数据中心能渲染：
   - 能看到 MT5 Import Center 或对应主要容器。
   - 自选列表表格区域存在。
   - StoreV5 面板区域存在。

3. SettingsPanel 懒加载可用：
   - 打开 settings tab。
   - 确认主要 tab 按钮存在。
   - 切换 tab 不报错。

4. StoreV5Panel CSS smoke：
   - 确认 `.ff-store-detail-table`、`.ff-store-direct-actions`、`.ff-chart-jump-controls` 这些关键类存在。
   - 截图或 DOM 尺寸断言，确保主要区域不是 0 高度。

5. ChartCoreHost smoke：
   - 图表容器存在。
   - canvas 或图表根元素不是空尺寸。

### 注意事项

1. E2E 不应依赖真实 MT5 终端。
2. 对网络请求做 mock 或允许后端不可用时仍能渲染空状态。
3. 不把截图作为唯一断言，优先 DOM 和尺寸断言。
4. 如果启动 dev server，需要脚本可重复运行，避免端口冲突。

### 验收标准

1. `npm run test:e2e` 通过。
2. `npm run test:logic` 通过。
3. `npm run build` 通过。
4. `.\scripts\check_all.ps1` 通过。
5. 文档记录手动回归尚未覆盖的真实 MT5 场景。
