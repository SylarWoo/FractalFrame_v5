你的逻辑可以定成：

```txt
摩根区间 V2 / 4H Core Cycle

核心区细分：8 等份
```

HLC3 也就是 Typical Price，标准计算就是 High、Low、Close 相加再除以 3；ATR 则是指定周期 True Range 的平均值，用来衡量波动率，所以你用 4H 的 ATR(7) 作为“当前 4H 波幅单位”是合理的。([富达][1]) 常见 Fibonacci 回撤层级里也包含 23.6%、38.2%、50%、61.8% 这些比例，所以你把 0→±1 的动态波幅域映射成 fib 层级，概念上就是“以波动率归一化后的斐波那契坐标系”。([图表学院][2])

## 1. 中心线计算

取**上一根已经收盘的 4H K线**：

```txt
H4_prev_high
H4_prev_low
H4_prev_close
```

中心线：

```txt
MC = (H4_prev_high + H4_prev_low + H4_prev_close) / 3
```

这里的 `MC` 就是当期 4小时摩根框架的中心位置。

## 2. 波幅单位计算

取 4H 周期上的 ATR(7)：

```txt
ATR7_4H = ATR(7) on 4H candles
```

然后定义完整摩根波幅半径：

```txt
R = 3 * ATR7_4H
```

所以：

```txt
上方 +1 = MC + R = MC + 3 * ATR7_4H
下方 -1 = MC - R = MC - 3 * ATR7_4H
```

也就是说：

```txt
MC 到 MC + 3ATR = 0 → +1
MC 到 MC - 3ATR = 0 → -1
```

## 3. Fibonacci 层级公式

统一公式可以写成：

```txt
Level(r) = MC + r * R
```

其中：

```txt
R = 3 * ATR7_4H
r ∈ [-1, +1]
```

比如：

```txt
+0.236 = MC + 0.236 * 3 * ATR7_4H
-0.236 = MC - 0.236 * 3 * ATR7_4H

+0.382 = MC + 0.382 * 3 * ATR7_4H
-0.382 = MC - 0.382 * 3 * ATR7_4H

+0.5 = MC + 0.5 * 3 * ATR7_4H
-0.5 = MC - 0.5 * 3 * ATR7_4H

+0.618 = MC + 0.618 * 3 * ATR7_4H
-0.618 = MC - 0.618 * 3 * ATR7_4H

+1 = MC + 3 * ATR7_4H
-1 = MC - 3 * ATR7_4H
```

## 4. 核心摩根区间

核心区域就是中心两个 0.236 中间：

```txt
Core_Low  = MC - 0.236 * R
Core_High = MC + 0.236 * R
```

因为：

```txt
R = 3 * ATR7_4H
```

所以也可以写成：

```txt
Core_Low  = MC - 0.236 * 3 * ATR7_4H
Core_High = MC + 0.236 * 3 * ATR7_4H
```

这个区域就是：

```txt
-0.236 → +0.236
```

它代表当前 4H 框架下的**核心均衡区 / 中心吸附区 / 震荡核心区**。

## 5. 核心区 8 等分

这次修正版：

```txt
核心区宽度 = Core_High - Core_Low
核心区分成 8 份
Core_Step = (Core_High - Core_Low) / 8
```

因为核心区从 `-0.236` 到 `+0.236`，总归一化宽度是：

```txt
0.236 - (-0.236) = 0.472
```

所以每一份在归一化坐标里是：

```txt
0.472 / 8 = 0.059
```

## 7. 解释口径

宝宝这套 4H 摩根区间可以这样定义：

```txt
4H 摩根区间以“上一根4H K线的 HLC3”为当前周期中心，
以“4H ATR(7) 的 3倍”为上下完整波幅边界，
把中心到上边界定义为 0→+1，

其中 -0.236 到 +0.236 是核心摩根区间，
核心区内部再分成 8 等份，
用于观察价格在中心区域内的吸附、偏移、震荡与突破准备。
```
